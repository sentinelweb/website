When seting up an existing Android project to use Jetpack compose - we want to re-use the standard MDC theme so that our compose views use the same theme coloring, typography and shapes as our standard Android views.

MDC-Android Compose Theme Adapter

The MDC-Android Compose Theme Adapter handles the import of the xml-based Android MDC Theme and lets you create a compose theme that looks the same. The library is part of the material-components library and you can use the same version as for compose:

implementation "com.google.android.material:compose-theme-adapter:$ver_compose

Creating the compose theme

When we use the theme we can 

@Composable
fun CuerTheme(
    darkTheme: Boolean = isSystemInDarkTheme(),
    content: @Composable () -> Unit,
) {
    val context = LocalContext.current
    val layoutDirection = LocalLayoutDirection.current
    var (colors, type, shapes) = createMdcTheme(
        context = context,
        layoutDirection = layoutDirection
    )

    MaterialTheme(
        colors = colors ?: throw IllegalArgumentException("Colors were not imported from MDC theme"),
        typography = type ?: throw IllegalArgumentException("Typography was not imported from MDC theme"),
        shapes = shapes ?: throw IllegalArgumentException("Shapes were not imported from MDC theme"),
        content = content
    )
}
There are a few things to note here:
  • This should be a function to ensure the correct theme is used at all times. There is not need to cache the returned theme object.
  • The values for (colors, type, shapes) are nullable, in the event that say shape attributes aren't set in your theme.xml then this value will be null so you can either make a default androidx.compose.material.Shapes object here (i.e. shapes ?: Shapes()) or throw an Exception as above to ensure fast-failure.
  • The darkTheme attribute can be used to override elements in the darlTheme as needed.

Using the compose theme

When we use the theme we can use it similarly to using a custom compose theme: 

@Composable
fun ArbitraryRootView(...) {
    CuerTheme {
        Surface {
            Text(
                text = "Hello world",
                style = MaterialTheme.typography.h1
            )
        }
    }
}
